Instructions

This practicum has a firm 120 minute time limit. The practicum is open book, open notes, open Internet, open everything – except help from your neighbors or instructor.

Problem

This exercise requires you to test and debug a partially completed movie list manager application written in Ruby. The application manages a personal list of movies the user may want to see. It provides support for ratings and reviews in helping to decide what movie to watch. The development team has written code to implement the following

  1. The movie manager maintains a collection of unique movie entries based on the name of the movie.
  2. Each movie entry contains the movie name, a numerical rating and optional text review.
  3. Movie entries may be created with movie names entered as either uppercase, lowercase or mixed-case strings.
  4. Movie names are stored entirely as uppercase strings.
  5. Valid ratings are 1 (worst) thru 10 (highest). If a rating is not indicated when adding a movie to the collection, or an invalid rating is entered, the rating defaults to 5.
  6. Reviews are entered in text format. If a review is not indicated when adding a movie to the collection, it defaults to “No Review Entered”.
  7. Movie entries may be deleted from the movie manager by specifying a movie name. If the movie is not found, the delete request is ignored.
  8. The movie manger can return a collection of movie names sorted alphabetically in ascending order.
  9. The movie manager can return a collection of movie entries sorted by rating in descending order.

 

None of the production code has been unit tested. There is no user interface for the initial release, so the strategy is to use Ruby’s unit testing framework to demonstrate that the first set of requirements have been correctly implemented. Your job is to write Ruby unit test cases for the classes that have been implemented and refactor the code should any defects be detected during your testing.

 

Instructions 

  1. Download the production code - MovieManager.rb and testing framework - TestMovieManager.rb
  2. Write the unit test cases in TestMovieManager required to meet the requirements stated above.
  3. You are free to modify the existing production code should defects arise.
  4. Do not change the signatures (names or calling parameters) of any existing production code methods.
  5. Do not implement additional functionality beyond what the requirements state.
  6. Should you uncover a defect – create a unit test that recreates the defect, and then perform any needed refactoring to fix the problem. Use method names in TestMovieManager that clearly identifiy the defect you are reproducing.
  7. Use the change logs in MovieManager.rb to identify any changes you made to the production code.
  8. Submit both MovieManager.rb and TestMovieManager.rb to a directory in your repsoitory named exactly practicum-4 . Do not change the names of the files.
  9. You are also being graded on the quality and thoroughness of your unit testing, not just correcting defects. Remember to test in small increments – one test at a time, and submit early and often. You should make multiple submissions; I will only grade the last one submitted.
