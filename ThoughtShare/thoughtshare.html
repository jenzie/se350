ThoughtShare Web Application
(15 Points)

The Technical Project (10 points)

Your task is to do some maintenance on ThoughtShare (a web application for thinkers to share their thoughts that is definitely not Twitter). ThoughtShare has the following entities:

  • Thinkers – the users of the system, who have names and URLs
  • Thoughts – these are short, interesting quotes shared by Thinkers
  • Follows – a relationship from one Thinker to another, so that Thinkers can only follow a small subset of their friends and colleagues
  • Thumb – a relationship from Thinker to Thought where one can “approve” of any given thought

Much of the application has been constructed for you, so your job will be to find bugs and fix them appropriately. For the final level of submission, you will need to add new functionality to the product.

You must complete and test your application at the levels in succession, but only your last submission at the highest level you attempt will be considered in grading your project.

The levels are outlined below, preceded by the number of points out that the level contributes to the 10 points available. Feel free to ask questions of your instructor if anything is unclear.

Level #0 Setup and Fix Misspelling
(1 point)

Download and unzip the source code for the app in a clean directory.

 

Important! We strongly recommend that you do this project in the lab, as the correct environment has been set up for this project there. If you do choose to attempt this on your home computer, be prepared to do some tricky setup of the correct version of Rails, Rake, and correct versions of the other gems needed.

First we need to setup our database.

 

> bundle exec rake db:setup

 

NOTE: The command for this version of Rails is slightly different than what we did in class. If you get a message about the wrong version of rake, then you didn’t do the “bundle exec” part. Also, do db:setup and not db:migrate.

 

Now seed the database with some initial data.

 

> bundle exec rake db:seed

 

As demonstrated in class, run the WebBrick server in Windows:

 

> rails server

 

Go to http://localhost:3000 to see the app in action.

 

To receive credit for this level, you must correct two spelling mistakes in the app. Both are apparent on the front page, but must be corrected in different places.

 

Level #1 Input Validation
(3 points)

 

For this level, you need to provide input validation in the following ways:

  • When sharing a new Thought:
    • The thought must have a minimum length of 5
    • The thought must have a maximum length of 154 (10% better than Twitter!!)
    • All thoughts must be unique (no two thinkers can have the same thought)
  • When creating a new Thinker:
    • The name must be between 1 and 35 characters
    • The name must be unique
    • The name can only include upper-case letters, lower-case letters, numbers, spaces, hyphens, and apostraphes. The name must start with an upper-case or lower-case letter.
    • The URL must match the following regex (use this snippet directly in your code):
      URI::regexp(%w(http https))
    • The URL must be unique
    • The URL must have a maximum of 120 characters

Useful documentation for this level: Validation in Rails

Level #3 Show Thumbers
(3 points)

On the main page, the green numbers next to the thumbs are actually a link to a page that doesn’t exist. Create this page to show all of the people who thumbed that thought. You will need to do a few things:

  • The name of the action is thumbers and is part of the thoughts view and controller, so...
    • You will need to add a thumbers method to the thoughts controller
    • You will need to add a file to the thoughts view
  • To determine how to get the ID of the thought being passed to your new page, check the server console right after you click on the link (look for the Parameters: { } hash being displayed)
  • In your config/routes.rb, you will need to add the following line after the resources line:
    match ':controller/:action/:id', :controller => :thoughts, :action => :thumbers

The page must have the following:

  • The page must have the same look and feel of the previous pages,
    • Including the bar at the top, without duplicating code.
    • The "active thinker" functionality should still work properly on this page (i.e. Logged in as)
  • The thought itself on the top of the page, inside of <h1> </h1> tags
  • The text "A thought by X"" where X is name of the thinker of that thought, inside <h3> </h3> tags
  • The text ""Thumbers of this thought” inside <h2> </h2> tags
  • A list of the thinkers who gave this thought at least one thumb. No repeated thinkers in this list.

Level #4 Add Attribute to a Model
(1 point)

Add a new string attribute called email to the Thinker model. You will must follow the Rails guidelines and conventions by doing the following:

  • Create a database migration using rails generate migration, using the conventional name for the migration: AddEmailToThinker
    • When you fill in the actual migration code – use the self.up and self.down methods – don’t add a change method as some documentation recommends (that only works in a future version of Rails).
  • The email must be validated using the following regex:
    /\A[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]+\z/
  • Add the email field to the New Thinker page
  • Show the thinker's email on their own profile page (when you click on a Thinker's name on the main page)

Useful documentation for this level: Database Migrations in Rails

Level #5 Thumb Uniqueness
(1 point extra credit)

Currently, a given thinker can thumb a thought multiple times. This is problematic because someone can artificially inflate one’s popularity of a thought. Add a validation constraint such that:

  • A thinker cannot thumb the same thought twice
  • If this is violated, the app should show nothing – that is, don’t show an error message or notify the user, but also don’t add the thumb.

Note: for the sake of this project, you don’t need to enforce this at the database level, only at the model level. But, in practice, you will also want to add a unique constraint in the Thumbs table since Rails model validations are not transaction-safe.
